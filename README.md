# AI+具身智能资讯聚合

> 面向人工智能与具身智能领域的自动化资讯聚合系统，覆盖算法突破、行业动态、政策信号、市场数据等多维度信息。

## 摘要

本项目实现了一套面向人工智能与具身智能领域的自动化资讯聚合管线（Pipeline），每日自动完成多源信息的采集、分类、评级与发布。系统从多个信息源抓取原始资讯，按照统一的数据模式进行结构化编码，经多维度分析框架处理后，生成静态网站并通过 GitHub Pages 发布。

系统每日定时运行（北京时间 18:00），执行以下三个阶段：

1. **数据采集与结构化** — 原始资讯经采集后编码为统一 JSON 格式，附加分类标签与质量评级。
2. **网站构建** — 基于最近 7 天数据自动生成响应式静态网页。
3. **发布与清理** — 生成内容提交并推送至 GitHub Pages；超过 7 天滚动窗口的历史数据自动清理。

## 一、系统架构

```
data/                       # 每日 JSON 数据文件（YYYY-MM-DD.json）
│
├── build.js                # 静态站点生成器（JSON → HTML）
├── publish.js              # 一键发布脚本（构建 + 清理 + 推送）
├── validate.js             # 数据质量校验工具（完整性、去重、字段检测）
├── cleanup.js              # 历史数据清理工具
│
├── site/                   # 生成的静态站点（GitHub Pages 根目录）
└── index.html              # 跳转入口 → site/index.html
```

## 二、数据模式

每日数据文件（`data/YYYY-MM-DD.json`）遵循如下结构：

```json
{
  "YYYY-MM-DD": {
    "update_time": "ISO 8601 时间戳",
    "hot_topics":      [ { "资讯条目" }, ... ],
    "startup":         [ { "资讯条目" }, ... ],
    "pm_view":         [ { "资讯条目" }, ... ],
    "algo_view":       [ { "资讯条目" }, ... ],
    "pm_project_view": [ { "资讯条目" }, ... ],
    "action_items":    [ { "资讯条目" }, ... ]
  }
}
```

每条资讯包含以下字段：

| 字段           | 类型     | 说明                                 |
|----------------|----------|--------------------------------------|
| `title`        | string   | 资讯标题                             |
| `source`       | string   | 信息来源（媒体、平台、机构）         |
| `url`          | string   | 原文链接（规范 URL）                 |
| `summary`      | string   | 资讯摘要                             |
| `category`     | string   | 内容分类（见第三节）                 |
| `rating`       | number   | 质量相关度评分（0–10）               |

## 三、内容分类体系

资讯按以下类别进行分类标注：

| 类别（英文标识）       | 类别名称     | 说明                                   |
|------------------------|-------------|----------------------------------------|
| `policy_signal`        | 政策信号     | 政府政策、监管动态、财政支持计划         |
| `algorithm_breakthrough` | 算法突破   | 模型发布、基准测试结果、学术论文         |
| `industry_dynamics`    | 企业动态     | 公司公告、战略合作、产品发布             |
| `market_data`          | 市场数据     | 融资轮次、估值变动、上市信息、市场分析   |
| `open_source`          | 开源动态     | 仓库发布、SDK 更新、社区活动             |
| `hardware`             | 硬件与机器人 | 机器人硬件、传感器、执行器、控制器       |
| `academic`             | 学术前沿     | 会议论文、预印本、实验室成果             |
| `other`                | 其他         | 未归入以上类别的资讯                     |

## 四、分析视角

本系统从五个分析维度对资讯进行组织与呈现：

| 视角               | 说明                                           |
|--------------------|------------------------------------------------|
| **热点聚焦**       | 跨类别筛选的高含金量每日热点                     |
| **创业板块**       | 具身智能赛道创业公司动态与择业价值判断           |
| **产品经理视角**   | 市场机会研判、标准化趋势、产品形态演进           |
| **算法工程师视角** | 技术路线分析、开源模型选型、算力方向             |
| **项目经理视角**   | 项目节奏、风险清单、跨团队协作                   |
| **行动建议**       | 技术选型建议、赛道判断、关注清单                 |

## 五、数据质量控制

### 5.1 数据校验

运行 `validate.js` 校验 JSON 完整性、检测重复条目、确认链接有效性、检查评分分布：

```bash
node validate.js
```

预期输出：

```
✅ JSON valid
Total: <总数>
Articles: <条目数>
Unique titles: <n> / <n>
All have URLs: true
Ratings range: <最小值> - <最大值>
Articles >= 10: PASS
No duplicates: PASS
```

### 5.2 质量约束

- 每日数据文件至少包含 10 条资讯（`PASS` 阈值）。
- 同一日期内资讯标题须唯一（不允许重复）。
- 所有资讯须包含有效的 `url` 字段。
- 评分值范围限制为 `[0, 10]`。

## 六、构建与发布管线

### 6.1 构建站点

基于最近 7 天数据生成静态站点：

```bash
node build.js
```

生成 `site/index.html`，功能包括：

- 按日期切换（最多 7 天）
- 按视角标签筛选
- 深色/浅色主题切换
- 评分星级显示
- 移动端响应式布局

### 6.2 一键发布

执行完整发布管线（构建 + 清理 + 推送）：

```bash
node publish.js
```

依次执行以下三步：

1. **构建** — 从 `data/*.json` 重新生成 `site/index.html`
2. **清理** — 移除超出 7 天滚动窗口的数据文件
3. **推送** — `git add .` → `git commit` → `git push origin main`

### 6.3 清理历史数据

手动清理过期数据：

```bash
node cleanup.js
```
