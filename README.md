# AI+具身智能资讯聚合

> 面向人工智能与具身智能领域的自动化资讯聚合系统，覆盖算法突破、行业动态、政策信号、市场数据等多维度信息。

## 摘要

本项目实现了一套面向人工智能与具身智能领域的自动化资讯聚合管线（Pipeline），每日自动完成多源信息的采集、分类、评级与发布。系统从多个信息源抓取原始资讯，按照统一的数据模式进行结构化编码，经多维度分析框架处理后，生成静态网站并通过 GitHub Pages 发布。

系统每日定时运行（北京时间 18:00），执行以下三个阶段：

1. **数据采集与结构化** — 原始资讯经采集后编码为统一 JSON 格式，附加分类标签与质量评级。
2. **网站构建** — 基于最近 7 天数据自动生成响应式静态网页。
3. **发布与清理** — 生成内容提交并推送至 GitHub Pages；超过 7 天滚动窗口的历史数据自动清理。

## 一、系统架构

```
data/                       # 每日 JSON 数据文件（YYYY-MM-DD.json）
│
├── build.js                # 静态站点生成器（JSON → HTML + RSS + Sitemap）
├── publish.js              # 一键发布脚本（构建 + 清理 + 推送）
├── validate.js             # 数据质量校验工具（完整性、去重、字段检测）
├── cleanup.js              # 历史数据清理工具
│
├── site/                   # 生成的静态站点（GitHub Pages 根目录）
│   ├── index.html          # 主站点
│   ├── feed.xml            # RSS 订阅源
│   ├── sitemap.xml         # 搜索引擎站点地图
│   └── data/               # 数据文件副本（供前端 AJAX 加载）
├── CONTRIBUTING.md         # 贡献指南
└── index.html              # 跳转入口 → site/index.html
```

## 二、数据模式

每日数据文件（`data/YYYY-MM-DD.json`）遵循如下结构：

```json
{
  "YYYY-MM-DD": {
    "update_time": "ISO 8601 时间戳",
    "hot_topics":      [ { "资讯条目" }, ... ],
    "startup_board":   { "companies": [...], "trend": "赛道趋势" },
    "pm_board":        { "insights": [...], "focus": [...] },
    "algo_board":      { "insights": [...], "tech_radar": [...], "focus": [...] },
    "projmgr_board":   { "insights": [...], "focus": [...] },
    "action_items":    { "tech_selection": [...], "track_judgment": [...], "watch_list": [...] },
    "raw_articles":    [ { "资讯条目" }, ... ],
    "hermes_board":    { "insights": [...], "tech_stack": [...], "community": [...] }
  }
}
```

每条资讯包含以下字段：

| 字段           | 类型     | 说明                                 |
|----------------|----------|--------------------------------------|
| `title`        | string   | 资讯标题                             |
| `source`       | string   | 信息来源（媒体、平台、机构）         |
| `url`          | string   | 原文链接（规范 URL）                 |
| `summary`      | string   | 资讯摘要                             |
| `category`     | string   | 内容分类（见第三节）                 |
| `rating`       | number   | 质量相关度评分（1–5，见评分标准）    |

## 三、内容分类体系

资讯按以下类别进行分类标注。数据文件中可使用中文标签（如"大模型"、"具身智能"），站点会自动映射为统一英文标识。

| 英文标识                | 类别名称      | 说明                                   | 常见中文标签            |
|-------------------------|--------------|----------------------------------------|-------------------------|
| `policy_signal`         | 政策信号      | 政府政策、监管动态、财政支持计划         | 政策信号、政策、监管      |
| `algorithm_breakthrough`| 算法突破      | 模型发布、基准测试结果、学术论文         | 大模型、算法突破、模型发布 |
| `industry_dynamics`     | 企业动态      | 公司公告、战略合作、产品发布             | 企业动态、合作、产品发布  |
| `market_data`           | 市场数据      | 融资轮次、估值变动、上市信息、市场分析   | 市场动态、融资、财报      |
| `open_source`           | 开源动态      | 仓库发布、SDK 更新、社区活动             | 开源动态、开源           |
| `hardware`              | 硬件与机器人  | 机器人硬件、传感器、执行器、控制器       | 具身智能、机器人、硬件    |
| `academic`              | 学术前沿      | 会议论文、预印本、实验室成果             | 学术前沿、论文、学术      |
| `other`                 | 其他          | 未归入以上类别的资讯                     | 其他                     |
| `大模型`              | 大模型/AI模型  | 替代 `algorithm_breakthrough`，指代更直观 |
| `具身智能`            | 具身智能/机器人 | 替代 `hardware`，行业通用术语              |
| `人形机器人`          | 人形机器人      | 具身智能子领域，特指双足机器人              |
| `市场动态`            | 市场数据/融资   | 替代 `market_data`，日常用语               |
| `政策`                | 政策信号        | `policy_signal` 的简称                      |
| `重要企业动态`        | 企业动态        | 强调重要性等级                              |
| `行业分析`            | 企业动态        | 行业研究机构发布的分析报告                  |
| `AI Agent`            | AI应用          | 编程Agent、智能助手等具体应用场景           |
| `AI应用`              | AI应用          | 泛指AI在各行业的落地应用                    |
| `行业展会`            | 企业动态        | 会议、博览会等活动类资讯                   |
| `开源项目`            | 开源动态        | 替代 `open_source`                          |
| `创业公司`            | 企业动态        | 初创企业相关动态                            |
| `学术论文`            | 学术前沿        | 替代 `academic`                             |

## 四、评分标准

每条资讯附带 1–5 分质量评级，评分标准如下：

| 评分 | 含义     | 说明                                         |
|------|----------|----------------------------------------------|
| 1    | 低质     | 来源不明、信息量低、纯转载无增量  |
| 2    | 一般     | 媒体通稿、常规报道、信息增量有限  |
| 3    | 中等     | 有一定行业参考价值、数据可靠      |
| 4    | 高质     | 独家信息、深度分析、权威来源      |
| 5    | 顶尖     | 行业里程碑事件、重大突破          |

站点中鼠标悬停在星级评分上可查看评分说明。

## 五、数据采集管线

### 5.1 采集流程

资讯采集通过 **Hermes Agent**（Nous Research 开源 AI 代理框架）驱动，执行以下步骤：

1. **信源扫描** — AI Agent 访问多个信息源（技术媒体、行业公众号、学术平台、社交媒体）
2. **内容提取** — 提取标题、摘要、链接、发布时间、来源
3. **智能分类** — AI 根据内容自动标注分类标签（category）
4. **质量评级** — AI 根据信息增量、来源权威性、行业重要性进行评分（rating 1–5）
5. **结构化输出** — 生成标准 JSON 数据文件存入 `data/` 目录

### 5.2 信源列表

当前覆盖信息源包括但不限于：
- **技术媒体**：TechXplore、MIT Technology Review、36氪、机器之心
- **财经媒体**：华尔街日报、第一财经、经济观察网、界面新闻
- **官方媒体**：人民网、China Daily、CGTN
- **行业平台**：赛塔网、多课网、投中网
- **学术平台**：arXiv RSS（计划中）、学术会议预印本

### 5.3 扩展信源

欢迎通过 Pull Request 或以 Issue 形式推荐新信源。参见 [CONTRIBUTING.md](CONTRIBUTING.md)。

## 七、Hermes 模块

本系统集成 **Hermes Agent**（Nous Research 开源 AI 代理框架）作为自动化驱动引擎，提供资讯采集、处理、发布的完整代理工作流。Hermes 模块在站点中新增独立标签页，展示以下内容：

| 子模块           | 说明                                               |
|------------------|----------------------------------------------------|
| **Hermes 资讯**  | Hermes Agent 最新动态、版本更新、社区讨论（含链接） |
| **技术栈与工具** | 框架、模型、工具链的最新资讯（含链接）             |
| **社区动态**     | 开源贡献、企业应用案例、生态建设进展（含链接）     |

数据存储在 `hermes_board` 字段中，支持每日更新。

## 八、数据质量控制

### 8.1 数据校验

运行 `validate.js` 校验 JSON 完整性、检测重复条目、确认链接有效性、检查评分分布：

```bash
node validate.js
```

预期输出：

```
✅ JSON valid
Total: <总数>
Articles: <条目数>
Unique titles: <n> / <n>
All have URLs: true
Ratings range: <最小值> - <最大值>
Articles >= 10: PASS
No duplicates: PASS
```

### 8.2 质量约束

- 每日数据文件至少包含 10 条资讯（`PASS` 阈值）。
- 同一日期内资讯标题须唯一（不允许重复）。
- 所有资讯须包含有效的 `url` 字段。
- 评分值范围限制为 `[1, 5]`。

## 九、构建与发布管线

### 9.1 构建站点

基于最近 7 天数据生成静态站点：

```bash
node build.js
```

生成文件：
- `site/index.html` — 主站点（含键盘导航、收藏、搜索等功能）
- `site/feed.xml` — RSS 2.0 订阅源
- `site/sitemap.xml` — 搜索引擎站点地图
- `site/data/*.json` — 数据文件副本

### 9.2 站点功能

- 按日期切换（最多 7 天）
- 按视角标签筛选（支持数字键 1-8 快速切换）
- 深色/浅色主题切换
- 评分星级显示（hover 可查看评分说明）
- 收藏/稍后阅读（localStorage 持久化）
- 键盘导航（↑↓ 浏览卡片，Enter 打开链接，Esc 关闭面板）
- 全文搜索（Ctrl+K）
- 分享和复制链接
- 移动端响应式布局

### 9.3 一键发布

执行完整发布管线（构建 + 清理 + 推送）：

```bash
node publish.js
```

依次执行以下三步：

1. **构建** — 从 `data/*.json` 重新生成 `site/index.html` + RSS + Sitemap
2. **清理** — 移除超出 7 天滚动窗口的数据文件
3. **推送** — `git add .` → `git commit` → `git push origin main`

### 9.4 清理历史数据

手动清理过期数据：

```bash
node cleanup.js
```

## 十、订阅方式

### RSS 订阅
在 RSS 阅读器中添加：`https://shnywang.github.io/ai_news/site/feed.xml`

### GitHub Watch
点击仓库右上角 Watch → 选择 "All Activity" 可获得更新通知。

### 邮件订阅（已上线）
站点底部提供邮件订阅入口，输入邮箱即可订阅每日资讯推送。

**订阅流程：**
1. 在站点底部输入邮箱并点击订阅
2. 系统发送确认邮件到您的邮箱
3. 每天 **北京时间 9:00** 自动收到当日资讯日报

**取消订阅：**
1. 已订阅用户点击"取消订阅"按钮
2. 系统发送取消确认邮件到管理员邮箱
3. 管理员确认后从订阅列表移除

**技术实现：**
- 订阅服务基于 [Formsubmit](https://formsubmit.co/) 实现，无需后端服务器
- 取消订阅由管理员脚本 `ai_news_unsubscribe_monitor.py` 自动处理
- 每日定时推送由 `send_newsletter.py` 脚本配合 Cron Job 完成

**当前订阅者邮箱存储在：** `subscribers.json`（仅管理员可访问）
